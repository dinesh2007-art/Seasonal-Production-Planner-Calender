# Backend Objectives

1. Create a working backend server with a health route that confirms the Seasonal Production Planning Calendar API is running.
2. Design API routes for creating, listing, viewing, and updating seasonal production planning records.
3. Validate business-critical fields including admin, production item, batches, upcoming festival, expected order volume, procurement timeline, and production capacity.
4. Build a capacity and utilisation tracking engine that calculates production days required, utilisation percentage, and risk level.
5. Prepare the backend structure so future database integration can store production planning, orders, and inventory records cleanly.
