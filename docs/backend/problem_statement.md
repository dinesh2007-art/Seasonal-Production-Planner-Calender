# Backend Problem Statement

Sharadha Stores needs a backend system for its Seasonal Production Planning Calendar because festival production planning depends on accurate data about expected orders, ingredient procurement, batch planning, and production capacity. At present, this work can become slow and error-prone when the admin manually maps expected order volumes to ingredient procurement timelines and production capacity.

From the backend perspective, the main problem is the absence of a central data model and API layer. Production plans, orders, inventory requirements, procurement dates, and capacity calculations need to be stored in a structured database and made available through reliable API routes.

The system must validate the data entered by staff, calculate whether production capacity can meet festival demand, and flag risky plans early. This reduces the chance of stockouts during peak demand and gives Sharadha Stores a dependable planning workflow.
